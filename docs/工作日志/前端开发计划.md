## 总体目标

将 `Frontend` 从当前可运行 MVP 深化为符合四份设计文档的完整前端：覆盖 FR1-FR6 的路网建模、节点微调、TM:PE 交规编辑、严格仿真、场景管理、评估可视化与报表导出。每个阶段都保持 `npm run type-check` 与 `npm run build` 通过。

## 推荐实施顺序

### 第一阶段：稳定 MVP 与文案清理

1. 修复历史中文乱码。（源码扫描未发现典型乱码，后续继续巡检图标/占位字符）
2. 补齐冲突提示 toast。（已完成轻量 typed notification）
3. 将所有已有可编辑操作纳入命令模式。（已完成右侧属性与信号灯编辑命令化；仿真参数和绘路首点清理后续单独推进）
4. 为道路升级增加“阻止降级冲突”选项。（已完成严格阻止 MVP，后续可接入统一 toast 展示详情）
5. 补充 Store/Command 单元测试。

### 第二阶段：FR1/FR2 建模闭环

1. 完整半透明道路预览。
2. 曲线/自由绘制。
3. 智能吸附、角度约束、测量标注。
4. 节点拖拽、Gizmo、多边形边界编辑。
5. 动态拓扑自愈。

### 第三阶段：FR3 交规闭环

1. 车道连接器 UI。
2. 车道箭头 UI 与 Decal 渲染。
3. 信控多步阶编辑器。
4. 转向限制、斑马线、禁行、标线完整编辑。
5. 规则校验服务与命令化。

### 第四阶段：FR4 仿真闭环

1. OD 泊松发车和默认随机流量。
2. A* 路径规划。
3. IDM 跟驰与红灯停车。
4. MOBIL 平滑变道与实线约束。
5. 车道连接器引导和车型禁行强校验。

### 第五阶段：FR5/FR6 场景与评估

1. 样例库一键加载。
2. 快照列表与回滚。
3. 热力图接入真实指标。
4. 交通路线飞线。
5. 车道指标、LOS、CSV/PDF 报表导出。

## 每阶段固定验收命令

```bash
npm run type-check
npm run build
```

通过后再进入下一阶段，避免大批量改动后难以定位问题。
